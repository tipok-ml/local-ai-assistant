# local-ai-assistant
# 🧠 Локальный AI-ассистент: как запустить LLM в России без дата-центра

Этот проект — пошаговое руководство и набор скриптов для развертывания полностью локального AI-ассистента (чат-бота, веб-интерфейса, Telegram-бота) на вашем собственном железе. Особое внимание уделено проблемам, с которыми сталкиваются пользователи из **России**: блокировки репозиториев, медленная загрузка, ограничения видеопамяти и т.д.

🔗 **Репозиторий содержит:**
- Подробный анализ проблем локального запуска LLM.
- Готовые скрипты для установки Ollama, Open WebUI, Telegram-бота.
- Примеры конфигурации для запуска на RTX 3060 Ti 8GB.

---

## 🚀 Возможности

- **Полностью локально** – все данные остаются на вашем ПК.
- **Работает без интернета** после загрузки моделей.
- **Поддержка GPU** (NVIDIA CUDA) – быстрая генерация.
- **Telegram-бот** – доступ с телефона.
- **Веб-интерфейс** (Open WebUI) – как у ChatGPT.
- **Веб-поиск** (DuckDuckGo) через внешний API.

## 🧩 Реальные проблемы, с которыми вы столкнётесь

<details>
<summary>🌐 Доступ к моделям (российская специфика)</summary>

- Блокировка Hugging Face, GitHub Container Registry.
- `ollama pull` выдаёт ошибки 401, 403 или бесконечно висит.
- Скорость загрузки 5–10 ГБ может занять часы.
- Некоторые модели скрыты для российских IP.

**Решение:** используйте зеркала (`hf-mirror.com`), качайте GGUF через браузер, настраивайте прокси для Ollama.
</details>

<details>
<summary>🖥️ Железо и видеопамять</summary>

- Модель должна целиком помещаться в VRAM для работы на GPU.
- RTX 3060 Ti (8GB) → модели 7B–8B (Q4_K_M).
- 14B модели уже с трудом, падает скорость.
- Модели 70B+ требуют серверных GPU и десятков тысяч долларов.

**Решение:** используйте квантованные GGUF-файлы (Q4_K_M), реалистично оценивайте возможности.
</details>

<details>
<summary>🧠 Производительность и CPU</summary>

- Если модель не влезает в VRAM – задействуется CPU (падение скорости в 10–30 раз).
- Веб-поиск (RAG) нагружает процессор и оперативную память.

**Решение:** для 80% задач достаточно моделей 7B–8B, не гонитесь за гигантами.
</details>

<details>
<summary>🔧 Технические сложности</summary>

- Драйверы NVIDIA под Linux (особенно в Kali) могут конфликтовать.
- Docker и `host.docker.internal` не всегда работают.
- Не все GGUF-файлы совместимы с Ollama.
- Установка Python-пакетов через `pip` в Kali ломает систему – нужен `pipx`.

**Решение:** доверяйте проверенным сборщикам (TheBloke, mradermacher), для Docker используйте `--network host`, пакеты ставьте через `pipx`.
</details>

<details>
<summary>🧪 Неожиданности во время работы</summary>

- Модели могут зацикливаться (`repeat_penalty`, `temperature`).
- Долгий первый запуск (30–60 секунд).
- Ограниченная документация для России.

**Решение:** настраивайте параметры в `Modelfile`, держите историю диалога короткой, делитесь опытом.
</details>

## 🛠️ Быстрая установка

### 1. Установите Ollama

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

Если из-за блокировок не качает – скачайте `.deb` вручную с [официального сайта](https://ollama.com/download/linux).

### 2. Загрузите модель (например, Qwen3.5 9B)

```bash
ollama run qwen3.5:9b-q4_K_M
```

При проблемах с загрузкой используйте зеркало:

```bash
ollama run hf.co/tensorblock/Qwen3-8B-GGUF:Q4_K_M
```

### 3. Запустите Telegram-бота (опционально)

```bash
git clone https://github.com/tipok-ml/local-ai-assistant
cd local-ai-assistant
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
# Замените TOKEN в telegram_bot.py на свой
python telegram_bot.py
```

### 4. Веб-интерфейс (Open WebUI)

```bash
docker run -d -p 8080:8080 --network host -v open-webui:/app/backend/data --name open-webui ghcr.io/open-webui/open-webui:main
```

Откройте `http://localhost:8080`, зарегистрируйтесь, в настройках укажите `http://localhost:11434` как Ollama Base URL.

### 5. Веб-поиск (External Search API)

```bash
python search_api.py &
```

В Open WebUI выберите `External Search API` и укажите `http://localhost:8000/search` и ключ `my_secret_token_123`.

## ⚙️ Настройка под ваше железо

- **8 ГБ VRAM (RTX 3060 Ti, RTX 4070)** → модели 7B–8B (Q4_K_M), например `qwen3.5:9b-q4_K_M`, `deepseek-r1:8b`, `white-rabbit-neo:7b`.
- **12 ГБ VRAM** → можно пробовать 14B модели (Q4_K_M).
- **24+ ГБ VRAM** → модели 32B–70B, Llama 4 Scout (17B активных), DeepSeek V4 Pro.

Полный список параметров и советов по оптимизации – в файле [CHALLENGES.md](CHALLENGES.md).
